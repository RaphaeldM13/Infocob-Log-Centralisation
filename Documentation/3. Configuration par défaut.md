# Configuration par défaut
La configuration suivante est la configuration en place après l'installation de loki et le déploiement de Alloy sur au moins une machine.

## Compatibilités :
Actuellement, alloy peut être déployé efficacement sur trois environnements :
- Linux : Il est capable de récupérer et de trier les logs systèmes et applications.
- Windows : Il est capable de récupérer et de trier les logs systèmes, applications et sécurité.
- Docker : Il est capable de récupérer les logs des autres containers du réseau de docker sur lequel il est déployé. Un container pouvant générer toute sorte de logs en fonction de son contenu, il n'y a pas encore de système de tri prédéfinis pour cette source.

## Tri des logs reçu :
Chaque logs reçu traverse un pipeline différents en fonction de sa source, et finit parsé avec les mêmes labels :
 - architecture : l'architecture de la machine dont proviens le log
 - filename : le fichier présent sur la machine source où se trouve le log. Windows n'a pas de filename car les logs sont directement récupérés sur windowsevent
 - hostname : le nom de la machine d'où proviens le log
 - levelText : Le niveau du log
 - os : Le système d'exploitation de la machine source
 - source : la provenance du log

En fonction de l'environnement, les logs peuvent obtenir des labels spécifiques.
  ### Apache
   - http_class : type de code http reçu
   - ip : ip de la machine ayant effectué une requête sur le server
   - method : requête effectuée
   - path : page dans laquelle la requête a été effectuée
   - protocol : (?)
   - status : code http de la requête
  ### Docker
   - service name : nom du container dont proviens le log. Exception : unknown_services est le container loki local dans lequel se trouve l'ensemble des logs récupérés
