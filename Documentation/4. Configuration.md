# 1. Configuration de Grafana
Grafana est principalement conçue pour être configurée via son interface utilisateur (UI).
Il est recommandé de modifier les fichiers de configuration manuellement uniquement si tu veux définir des paramètres persistants sur plusieurs déploiements (ex : SMTP, provisioning, alerting, etc.).

  ## 1.1 SMTP
  
  La configuration du SMTP s'effectue dans le fichier [.env](Config/.env) :
    - `GF_SECURITY_ADMIN_PASSWORD` : Mot de passe utilisé pour se connecter à l'UI Grafana en local
    - `GF_SMTP_HOST` : Type d'adresse mail
    - `GF_SMTP_FROM_ADDRESS` : Adresse email utilisée pour envoyer les notifications
    - `GF_SMTP_PASSWORD` : Mot de passe de connexion à l'adresse d'envoie des notifications, générée par le fournisseur.
    - `GF_SMTP_USER` : Adresse de réception des notifications
    
  ## 1.2 Datasources

  La configuration des Datasources s'effectue sur Grafana, `/connections/datasources`, sauf si elles sont fournies en provisioning, auquel cas elles devront êtres configurées dans le fichier [contact-points.yaml](Config/provisioning/alerting/contact-point.yaml). Vous pouvez tout de même le configurer via Grafana si vous activez `editable: true`
  Faites attentions à ce que la valeur du header http reste identique à celui confiuré dans [alloy-local-config.yaml](Config/alloy-local-config.yaml)
